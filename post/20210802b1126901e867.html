natcat

2021-08-02

awa，没啥说的，这个工具几乎啥都能干（加密？不存在的）……

上参数表：

  • -v：显示所有过程
  • -n: 不对输入的IP进行DNS解析
  • -g <网关> 设置路由器跃程通信网关，最多可设置8个；
  • -G <指向器数目> 设置来源路由指向器，其数值为4的倍数；
  • -h 在线帮助；
  • -i <延迟秒数> 设置时间间隔，以便传送信息及扫描通信端口；
  • -l 使用监听模式，管控传入的资料；
  • -o <输出文件> 指定文件名称，把往来传输的数据以16进制字码倾倒成该文件保存；
  • -p <通信端口> 设置本地主机使用的通信端口；
  • -r 乱数指定本地与远端主机的通信端口；
  • -s <来源位址> 设置本地主机送出数据包的IP地址；
  • -u 使用UDP传输协议；
  • -w <超时秒数> 设置等待连线的时间；
  • -z 使用0输入/输出模式，只在扫描通信端口时使用。
  • -e 连接之后执行程序

举个栗子:

监听端口444，接受到连接后把自己的shell给对面

nc -l -p 444 -e /bin/bash

连接到192.168.1.1:444上监听的nc

nc -v 192.168.1.1 444

传输文件

nc -l -p 444 > awa.txt #接收 nc -v 192.168.1.1 < awa.txt #发送
